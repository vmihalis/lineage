    **1. Create `src/display/event-renderer.ts`:**

    The EventRenderer class manages four concerns:
    A) Event subscription/unsubscription with safe handler tracking
    B) Formatted console output for each event
    C) Spinner lifecycle around long-running operations
    D) Generation display state accumulation for summary tables

    ```typescript
    import ora from 'ora';
    import type { Ora } from 'ora';
    import { lineageBus } from '../events/index.js';
    import type { SimulationParameters } from '../schemas/index.js';
    import {
      formatBirth, formatDeath, formatTransmission, formatMutation,
      formatGenerationStart, formatGenerationEnd, formatInheritance,
      formatSimulationStart, formatSimulationEnd,
    } from './formatters.js';
    import {
      buildGenerationSummary, createGenerationDisplayState,
    } from './generation-summary.js';
    import type { GenerationDisplayState } from './generation-summary.js';
    ```

    Class structure:

    ```typescript
    export class EventRenderer {
      private spinner: Ora | null = null;
      private currentGenState: GenerationDisplayState | null = null;

      // Store bound handlers for safe detach (NEVER use removeAllListeners)
      private handlers = new Map<string, (...args: any[]) => void>();
    ```

    **attach() method:**
    Create named bound handlers for each event. Store them in the handlers Map. Subscribe each to lineageBus.on(). Events to subscribe:
    - `simulation:started` -> log formatSimulationStart(seedProblem)
    - `simulation:ended` -> stop spinner, log formatSimulationEnd(generationCount)
    - `generation:started` -> reset currentGenState via createGenerationDisplayState(generationNumber), log formatGenerationStart, start spinner "Citizens interacting..."
    - `citizen:born` -> stop spinner, log formatBirth, record in currentGenState.citizens Map, restart spinner
    - `citizen:died` -> stop spinner, log formatDeath, update citizen in currentGenState with deathProfile, restart spinner
    - `citizen:peak-transmission` -> stop spinner, log formatTransmission, mark citizen transmitted in currentGenState. Cross-reference: check if this citizen's transmissionId appears in mutatedTransmissions Map -- if so, set citizen's mutationType. Restart spinner.
    - `transmission:mutated` -> stop spinner, log formatMutation, record transmissionId -> mutationType in currentGenState.mutatedTransmissions. Restart spinner.
    - `inheritance:composed` -> stop spinner, log formatInheritance, restart spinner
    - `generation:ended` -> stop spinner, log buildGenerationSummary(currentGenState) if currentGenState exists, log formatGenerationEnd

    **Spinner management pattern:**
    Before any console.log in an event handler: `this.stopSpinner()` (checks if spinner exists and is spinning).
    After logging: `this.startSpinner(text)` (creates new spinner or updates text).
    Helper methods:
    ```typescript
    private log(message: string): void {
      this.stopSpinner();
      console.log(message);
    }

    private stopSpinner(): void {
      if (this.spinner) {
        this.spinner.stop();
        this.spinner = null;
      }
    }

    private startSpinner(text: string): void {
      this.stopSpinner();
      this.spinner = ora(text).start();
    }
    ```

    **detach() method:**
    Iterate over handlers Map. For each entry, call `lineageBus.removeListener(eventName, handler)`. Clear the Map. Stop any active spinner. This is SAFE -- it only removes display handlers, not subsystem listeners.

    **Generation state accumulation (for EVNT-03 summary):**
    - On `generation:started`: create new GenerationDisplayState via createGenerationDisplayState(generationNumber)
    - On `citizen:born`: add DisplayCitizen to currentGenState.citizens Map (keyed by citizenId)
    - On `citizen:died`: look up citizen in Map, set deathProfile
    - On `citizen:peak-transmission`: look up citizen in Map, set transmitted = true. Also cross-reference: check if this citizen's transmissionId appears in mutatedTransmissions Map -- if so, set citizen's mutationType.
    - On `transmission:mutated`: record transmissionId -> mutationType in currentGenState.mutatedTransmissions. Note: transmission:mutated fires BEFORE citizen:peak-transmission for the same citizen (per generation-runner.ts). The cross-reference happens in the citizen:peak-transmission handler.
    - On `generation:ended`: call buildGenerationSummary(currentGenState), log the table
