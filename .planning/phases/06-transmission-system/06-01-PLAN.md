    - extractAnchorTokens("[1] Single claim") returns ["Single claim"]
    - extractAnchorTokens("[1] Claim one\n[2] Claim two\n[3] Third claim\n[4] Fourth\n[5] Fifth") returns 5 elements
    - extractAnchorTokens("[10] Double digit works\n[11] Also works") returns ["Double digit works", "Also works"]

    - extractAnchorTokens("") returns [] (empty input edge case)
    - extractAnchorTokens("[1]  Extra spaces  \n[2] Normal") returns ["Extra spaces", "Normal"] (trimmed)
    - extractAnchorTokens("[1] Multi-line claim\nthat spans two lines\n[2] Next claim") extracts both claims with the continuation text
  </behavior>
  <implementation>
    Create src/transmission/anchor-parser.ts with a single exported function:

    ```typescript
    export function extractAnchorTokens(text: string): string[]
    ```

    Use regex `/\[(\d+)\]\s*(.+?)(?=\n\[\d+\]|\n*$)/gs` to match `[N] claim text` patterns.
    The regex captures the claim text (group 2) and handles multi-digit numbers.
    Trim each captured claim text.
    Skip empty captures.
    Fallback: if no `[N]` patterns found and input is non-empty, return `[text.trim()]`.
    Return empty array only for empty/whitespace-only input.

    File must use `.js` extension for imports (ESM nodenext resolution).
    No dependencies beyond the text parameter.
  </implementation>
</feature>
